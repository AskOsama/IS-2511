1. Overview

Security Requirements

Organizations need to protect their information assets from various threats. This requires understanding what assets need protection, what threats exist, and what countermeasures can be applied.

Key Questions to Consider

  • What assets need protection? - Data, systems, networks, people
  • What are the threats? - Hackers, malware, natural disasters, insider threats
  • What countermeasures exist? - Technical controls, policies, training
  • How do we assess risk? - Identify vulnerabilities, estimate impact, prioritize

Key Insight: Effective security management requires a systematic approach that considers people, processes, and technology working together.

2. IT Security Management

What is IT Security Management?

IT Security Management is the process of managing and protecting information assets through policies, procedures, and controls to ensure the confidentiality, integrity, and availability of information.

The CIA Triad

The three core objectives of information security:

Integrity
Availability

Confidentiality

Ensuring that information is accessible only to those authorized to access it.

  • Encryption: AES-256, RSA, TLS/SSL for data protection
  • Access controls: Role-based access (RBAC), least privilege principle
  • Authentication: Multi-factor authentication (MFA), biometrics
  • Data classification: Public, Internal, Confidential, Restricted

Risk Management Example: Protecting risk assessment reports from unauthorized access to prevent exposure of organizational vulnerabilities.

Integrity

Safeguarding the accuracy and completeness of information and processing methods.

  • Hash functions: SHA-256, SHA-3 for data verification
  • Digital signatures: Non-repudiation and authenticity
  • Version control: Git, document management systems
  • Audit trails: Logging all data modifications

Risk Management Example: Ensuring audit logs and compliance reports cannot be falsified or tampered with.

Availability

Ensuring that authorized users have access to information and associated assets when required.

  • Redundancy: RAID, clustering, load balancing
  • Backup systems: 3-2-1 rule (3 copies, 2 media types, 1 offsite)
  • Disaster recovery: RTO and RPO planning
  • DDoS protection: CDN, rate limiting, traffic scrubbing

Risk Management Example: Keeping security monitoring systems operational during an active attack.

IT Security Management Functions

  • Risk assessment and management
  • Security policy development
  • Security awareness training
  • Incident response
  • Compliance monitoring
  • Security architecture design

3. ISO 27000 Security Standards

What is ISO 27000?

The ISO/IEC 27000 family of standards helps organizations keep information assets secure. It includes standards for Information Security Management Systems (ISMS).

Key Standards in the ISO 27000 Family

Standard Description Key Focus
ISO 27001 Requirements for ISMS - the main certification standard Certification, requirements
ISO 27002 Code of practice for information security controls 93 controls, best practices
ISO 27005 Information security risk management Risk assessment methodology
ISO 27017 Security controls for cloud services Cloud-specific controls
ISO 27018 Protection of personal data in the cloud Privacy, PII protection
ISO 27701 Privacy information management extension to ISO 27001 GDPR alignment, privacy
ISO 27035 Information security incident management Incident response

ISO 27001:2022 Update: The latest version reorganizes controls into 4 themes (Organizational, People, Physical, Technological) with 93 total controls, including 11 new controls for emerging threats like threat intelligence and cloud security.

Information Security Management System (ISMS)

An ISMS is a systematic approach to managing sensitive company information so that it remains secure. It includes:

  • People and their roles
  • Processes and procedures
  • Technology and systems
  • Policies and documentation

Benefits of ISO 27001 Certification: Demonstrates commitment to security, builds customer trust, ensures legal compliance, and provides a framework for continuous improvement.

4. IT Security Management Process

The IT security management process consists of five key steps that form a continuous cycle. This framework aligns with the NIST Cybersecurity Framework used widely in industry:

The 5-Step Security Process

1

Identify

Assets, threats, vulnerabilities

2

Protect

Implement security controls

3

Detect

Monitor and analyze

4

Respond

Contain and take action

5

Recover

Restore and improve

Step 1: Identify

Develop organizational understanding to manage security risk to systems, assets, data, and capabilities.

  • Asset management and inventory
  • Business environment analysis
  • Governance and risk assessment
  • Threat identification

Step 2: Protect

Implement appropriate safeguards to ensure delivery of critical services.

  • Access control mechanisms
  • Security awareness training
  • Data security measures
  • Protective technology deployment

Step 3: Detect

Implement appropriate activities to identify the occurrence of a security event.

  • Continuous monitoring
  • Anomaly detection
  • Security event logging
  • Alert analysis

Step 4: Respond

Take action regarding a detected security incident.

  • Response planning
  • Communications
  • Analysis and containment
  • Mitigation activities

Step 5: Recover

Maintain plans for resilience and restore any capabilities impaired by a security incident.

  • Recovery planning
  • System restoration
  • Improvements based on lessons learned
  • Communications with stakeholders

5. Plan-Do-Check-Act Cycle

The Deming Cycle (PDCA)

The Plan-Do-Check-Act cycle is a four-step management method used for continuous improvement of processes and products. It is fundamental to ISO 27001 implementation and ensures security management is never a "set and forget" activity.

PDCA Continuous Improvement Cycle

PLAN

Define objectives
Design controls

DO

Implement controls
Execute plans

CHECK

Monitor results
Audit performance

ACT

Improve processes
Correct issues

Try Interactive PDCA Demo

PLAN Phase

Establish the objectives and processes necessary to deliver results in accordance with the expected output.

  • Define the ISMS scope
  • Establish security policy
  • Conduct risk assessment
  • Select controls to address risks

DO Phase

Implement the plan, execute the process, make the product.

  • Implement security controls
  • Develop procedures
  • Train personnel
  • Manage operations

CHECK Phase

Study the actual results and compare against expected results to determine any differences.

  • Monitor and measure processes
  • Conduct internal audits
  • Review ISMS effectiveness
  • Report findings to management

ACT Phase

Take corrective actions on significant differences between actual and planned results.

  • Implement corrective actions
  • Address non-conformities
  • Make improvements
  • Update documentation

Continuous Improvement: The PDCA cycle is not a one-time process. It should be repeated continuously to achieve ongoing improvement in security management.

6. Security Policy

What is a Security Policy?

A security policy is a document that outlines how an organization protects its information assets and guides decision-making regarding security.

Security Policy Topics

Key Areas to Address

  • Scope and objectives
  • Roles and responsibilities
  • Risk management approach
  • Asset classification
  • Access control requirements
  • Incident response procedures
  • Compliance requirements
  • Training and awareness
  • Business continuity
  • Review and update schedule

Example Security Policy Structure

Policy Document Components

  1. Purpose: Why the policy exists
  2. Scope: What/who the policy covers
  3. Policy Statement: The actual rules and requirements
  4. Responsibilities: Who is responsible for what
  5. Compliance: How compliance is monitored
  6. Exceptions: Process for handling exceptions
  7. Review: When and how the policy is reviewed

Common Policy Mistakes

  • Policies that are too vague or too detailed
  • Policies that don't align with business objectives
  • Policies that are never reviewed or updated
  • Policies without enforcement mechanisms

7. Management Support

Why Management Support Matters

Without senior management commitment, security initiatives often fail. Management provides the resources, authority, and organizational culture needed for effective security.

Senior Management Responsibilities

  • Approve and endorse security policies
  • Allocate adequate resources (budget, personnel)
  • Define security roles and responsibilities
  • Review security reports and metrics
  • Ensure security aligns with business objectives
  • Promote a security-aware culture

The IT Security Officer Role

The IT Security Officer (or Chief Information Security Officer - CISO) is responsible for:

  • Developing and maintaining security policies
  • Managing security operations
  • Coordinating security awareness programs
  • Reporting to senior management
  • Ensuring compliance with regulations
  • Managing incident response

Top-Down Approach: Security must be championed from the top. When executives visibly support security initiatives, the entire organization follows.

Summary and Key Takeaways

What We've Learned

  1. IT Security Management is a systematic approach to protecting information assets through policies, procedures, and controls.
  2. The CIA Triad (Confidentiality, Integrity, Availability) forms the foundation of all security objectives.
  3. ISO 27000 standards provide a framework for implementing an Information Security Management System (ISMS).
  4. The 5-Step Security Process - Identify, Protect, Detect, Respond, Recover - provides a practical framework for security operations.
  5. The PDCA Cycle (Plan-Do-Check-Act) enables continuous improvement in security management.
  6. Security Policies document how an organization protects its assets and guides security decisions.
  7. Management Support is essential for security success - without it, security initiatives will fail.
  8. The IT Security Officer plays a critical role in coordinating security efforts across the organization.
